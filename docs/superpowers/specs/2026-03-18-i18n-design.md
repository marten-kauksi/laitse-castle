# i18n Design — Laitse Loss Website

**Date:** 2026-03-18
**Status:** Approved

## Overview

Add Estonian + English language support to the Laitse Loss website using Astro's built-in i18n routing.

## URL Structure

| Page | Estonian (default) | English |
|------|-------------------|---------|
| Home | `/` | `/en/` |
| History | `/ajalugu` | `/en/history` |
| Events & Seminars | `/peod-ja-seminarid` | `/en/events-and-seminars` |

Redirect pages (`/peod`, `/seminarid`, `/sundmused`) remain Estonian-only redirects.

## Translation System

- **Translation files:** `src/i18n/et.json` and `src/i18n/en.json`
- **Helper:** `src/i18n/utils.ts` — exports `t(key)`, `getLocale()`, `getLocalizedPath()`
- All hardcoded UI strings extracted to translation files
- Pages pull text from translation files based on current locale
- Schema.org / SEO metadata also locale-aware
- English JSON starts as copy of Estonian (placeholder) until real English copy arrives

## Language Switcher

- Flag icons (Estonian + UK) with "ET" / "EN" text labels
- Placed in header
- Links to equivalent page in the other language
- Active language visually highlighted

## SEO

- `<html lang>` attribute set dynamically per locale
- `<link rel="alternate" hreflang>` tags on every page
- Sitemap includes both locale variants
- `og:locale` set per page

## Routing Approach

Use Astro's built-in i18n config in `astro.config.mjs`:
- `defaultLocale: 'et'`
- `locales: ['et', 'en']`
- `routing: { prefixDefaultLocale: false }` — Estonian at root, English under `/en/`

## Page Structure

Estonian pages stay at `src/pages/` (e.g., `index.astro`, `ajalugu.astro`).
English pages at `src/pages/en/` (e.g., `en/index.astro`, `en/history.astro`).
English pages import the same components, just pass `locale="en"` to pull English strings.

## What Doesn't Change

- Visual design, components, styling
- Image assets
- Page structure and layout
